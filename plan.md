# Plan

## Goal
Модернизировать сайт так, чтобы он выглядел профессионально, работал быстро, имел удобную фиксированную верхнюю навигацию на desktop и mobile, не перекрывал контент, корректно вел пользователя по разделам и содержал все ранее согласованные улучшения.

## Agreed Additions
- Вернуть и сохранить раздел `Главная`.
- Сделать возврат на главную из любого раздела.
- Убрать лишний брендинг/надписи из верхней части главной.
- Убрать цифры возле `Избранное` и `История`.
- Добавить копирайт внизу: `© PuMa10w with ❤️`.
- На мобильной версии верхнее меню должно быть:
  - фиксированным,
  - удобным для пальца,
  - по центру,
  - без перекрытия карточек и контента.

## Current Problems
1. Фиксированная шапка компенсируется ручными отступами и периодически перекрывает контент.
2. На мобильных навигация визуально не центрирована и неудобна.
3. Шапка перегружена и структурно нестабильна.
4. Главная и каталог визуально ощущаются как разные интерфейсы.
5. В коде есть рассинхрон между текущим UX и компонентами навигации.
6. Данные для `home` и переходов между разделами загружаются не самым надежным образом.

## Target UX
1. Чистая главная страница с полезными точками входа.
2. Компактная фиксированная шапка.
3. На desktop:
   - основная навигация и действия визуально выровнены,
   - контент всегда стартует ниже шапки.
4. На mobile:
   - меню фиксировано,
   - оба ряда по центру,
   - есть удобный горизонтальный скролл,
   - кнопки имеют удобную высоту,
   - контент не уезжает под header.
5. Внизу корректный footer с копирайтом.

## Implementation Plan

### 1. Refactor Header Layout
Files:
- `src/components/Navbar.tsx`
- `src/App.css`

Tasks:
1. Упростить структуру `Navbar.tsx`.
2. Удалить старый бренд-блок слева сверху.
3. Разделить header на два ряда:
   - primary row: `Главная`, `Гинекология`, `Акушерство`
   - utility row: `Опросники`, `Фармакология`, `Избранное`, `История`, `Тема`
4. Удалить счетчики у `Избранное` и `История`.
5. Упростить пропсы `Navbar`, убрать ненужные `favoritesCount` и `historyCount`.
6. Привести aria-атрибуты к корректному виду.

### 2. Build Stable Fixed Navigation
Files:
- `src/App.css`

Tasks:
1. Перестроить `.navbar` с текущей grid-модели на более предсказуемую вертикальную структуру.
2. Ввести стабильную схему offset для контента через CSS variables.
3. Убрать ручные хаотичные `margin-top` и привязать основной контент к высоте шапки.
4. Снизить визуальную тяжесть шапки:
   - уменьшить blur,
   - уменьшить тени,
   - уменьшить лишние радиусы/внутренние отступы.
5. На desktop сохранить фиксированность, но не допускать перекрытия карточек.

### 3. Make Mobile Header Centered and Usable
Files:
- `src/App.css`
- `src/components/Navbar.tsx`

Tasks:
1. Сделать на мобильных два фиксированных ряда внутри header.
2. У каждого ряда:
   - `display: flex`,
   - `justify-content: center`,
   - `overflow-x: auto`,
   - скрытый scrollbar.
3. Добавить внутреннюю scroll-обертку для визуально центрированной группы кнопок.
4. Настроить touch-friendly размеры:
   - высота кнопок не меньше `44px`,
   - лучше `46-48px`.
5. Уменьшить ширину и padding кнопок без потери удобства.
6. Проверить, что при узкой ширине экранов навигация остается удобной и не "уезжает" влево.
7. При необходимости добавить `scroll-snap-type: x proximity`.

### 4. Rebuild Content Offset System
Files:
- `src/App.tsx`
- `src/App.css`

Tasks:
1. Обернуть основной контент в `main.page-content`.
2. Вместо индивидуальных отступов у `search`, `cards`, `home` использовать единый верхний `padding-top`.
3. Привязать этот `padding-top` к фактической высоте фиксированной шапки.
4. Проверить, что:
   - поиск,
   - фильтры,
   - сетка карточек,
   - главная
   не перекрываются header ни на одном брейкпоинте.
